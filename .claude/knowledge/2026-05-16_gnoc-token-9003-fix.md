# KI: Xử lý lỗi 401 và Token trên cổng 9003

## 🔴 Problem
Token bắt được từ Portal (cổng 9000) đôi khi không có quyền truy cập API SR (cổng 9003), dẫn đến lỗi 401 dù Token vẫn còn hạn.

## 🔍 Root Cause
Hệ thống GNOC sử dụng các microservices khác nhau. Token có thể hợp lệ ở Gateway chính nhưng chưa được phân quyền hoặc chưa kích hoạt phiên làm việc cho dịch vụ SR cụ thể.

## ✅ Solution
- Chuyển việc kiểm tra Token (check_jwt) trực tiếp qua cổng 9003.
- Sử dụng cơ chế `force_login=True` để xóa Token cũ và bật Chrome ngầm lấy Token mới ngay khi gặp lỗi 401.

## 🛡️ Prevention
Luôn validate Token bằng một request "thử nghiệm" (pageSize=1) tới chính API nghiệp vụ trước khi bắt đầu tải dữ liệu lớn.
