<ai_system_protocol>
<purpose>
Tiêu chuẩn hóa quy trình cài đặt và đồng bộ định tuyến Kỹ năng Cốt lõi (Core Skills) cho AI Assistant (Antigravity/Cursor/Windsurf) thông qua nguồn lưu trữ Github. File này được thiết kế để AI trực tiếp đọc, hiểu bộ lệnh và thực thi bằng công cụ nội tại, không dành cho người đọc.
</purpose>

<strict_rules>

- TUYỆT ĐỐI KHÔNG DÙNG lệnh Copy/Paste file. Luôn sử dụng cơ chế liên kết ảo (Symbolic Link / Junction).
- KHÔNG tạo file dư thừa trong thư mục hệ thống.
- TỰ ĐỘNG nhận diện Hệ Điều Hành (OS) để đưa ra lệnh Command Line phù hợp.
  </strict_rules>

<prerequisites>
Sử dụng run_command chạy lệnh `git --version` để kiểm tra. Nếu hệ thống báo lỗi không nhận diện được git do chưa cài đặt, BUỘC PHẢI DỪNG QUY TRÌNH ngay lập tức và thông báo cho người dùng: "Hệ thống của bạn chưa cài đặt Git. Vui lòng cài đặt Git (https://git-scm.com/) trước khi chạy Protocol này".
</prerequisites>

<execution_workflow>

### Bước 1: Khởi tạo Storage Cache

1. Xác định thư mục cài đặt Global Skills của AI trên máy Host. Đường dẫn thường là:
   - MacOS / Linux: `~/.gemini/antigravity`
   - Windows: `$env:USERPROFILE\.gemini\antigravity`
2. Tại thư mục cha này, kiểm tra sự tồn tại của thư mục `_cached_repos`. Nếu chưa có, hãy sử dụng công cụ tạo thư mục này. Thư mục này nằm cùng cấp với thư mục `skills`.

### Bước 2: Tải/Đồng Bộ Source Code

1. Truy cập vào `_cached_repos` bằng `run_command`.
2. Thực thi xử lý 2 kho mã nguồn sau:
   - `https://github.com/sickn33/antigravity-awesome-skills`
   - `https://github.com/abhigyanpatwari/GitNexus`
3. Logic xử lý:
   - Nếu repo chưa tồn tại: Chạy `git clone [url]`.
   - Nếu repo đã tồn tại: Truy cập thẳng vào repo `cd [repo_name]` và chạy `git pull` để đồng bộ bản mới nhất.

### Bước 3: Đánh chỉ mục (Indexing) và Tự động Lọc Skill phù hợp

Thay vì sử dụng danh sách cố định, AI thực hiện quy trình phân tích và lựa chọn động để tối ưu cho từng dự án:

1. **Phân tích Project:** Quét các file chính trong dự án (ví dụ: `package.json`, `index.html`, `requirements.txt`, các file code logic) để nhận diện Tech Stack (Ngôn ngữ, Framework, UI/UX Style) và mục đích của dự án.
2. **Quét Kho Skill:** Duyệt danh sách các thư mục con trong `_cached_repos/antigravity-awesome-skills/skills` và `_cached_repos/GitNexus`.
3. **Tuyển chọn Skill (Selection):**
   - **Nhóm Bắt buộc (Tư duy & Quy trình):** Luôn phải có `clean-code`, `systematic-debugging`, `concise-planning`, `git-pushing`, `lint-and-validate`.
   - **Nhóm GitNexus:** Luôn bao gồm toàn bộ các thư mục có tiền tố `gitnexus-*`.
   - **Nhóm Phụ trợ Dự án:** Chọn thêm các skill khớp nhất với Tech Stack đã phân tích ở bước 1. Ví dụ: Dự án Web/JS chọn `javascript-mastery`, `antigravity-design-expert`; Dự án Python chọn `python-pro`, `fastapi-pro`, v.v.
4. **Giới hạn:** Duy trì tổng số skill cài đặt trong khoảng 15-20 để đảm bảo AI hoạt động tập trung và hiệu quả nhất.

### Bước 4: Tạo Symbolic Links

1. Điều hướng về đích: Thư mục `skills` Global của AI (ví dụ: `~/.gemini/antigravity/skills`).
2. Sử dụng `run_command`, lặp qua danh sách các đường dẫn Tuyển chọn ở Bước 3 và tạo liên kết:
   - Nếu Host OS là Windows (Powershell): Dùng lệnh `New-Item -ItemType Junction -Path ".\tên-skill" -Target "đường-dẫn-absolute-từ-bước-3"`
   - Nếu Host OS là Mac/Linux: Dùng lệnh `ln -s "đường-dẫn-absolute-từ-bước-3" "./tên-skill"`

### Bước 5: Verify & Report

1. Dùng lệnh xuất trạng thái (tree command hoặc dir) tại thư mục `skills` gốc.
2. In báo cáo tóm tắt trên khung chat để User nghiệm thu việc cài đặt hoàn tất dưới dạng cấu trúc cây (Tree graph) Text Base đơn giản. Báo cáo này phải show được Link Target đang trỏ chuẩn xác về `_cached_repos`.

### Bước 6: Khởi tạo Orchestration Guide (AGENTS.md)

Để AI có thể sử dụng hiệu quả bộ skill đã cài đặt, hệ thống thực hiện tạo file điều phối tại gốc dự án:

1. **Khởi tạo file:** Tạo file `AGENTS.md` tại thư mục gốc của project (nếu đã có thì cập nhật nội dung mới).
2. **Cấu trúc nội dung:**
   - **Project Context:** Tóm tắt Tech Stack và mục tiêu của dự án (đã phân tích ở Bước 3).
   - **Active Agents/Skills:** Liệt kê danh sách các skill vừa cài đặt.
   - **Role & Capability:** Với mỗi skill, AI phải tóm tắt 1 câu về năng lực cốt lõi của nó dựa trên việc đọc file `SKILL.md` tương ứng.
   - **Coordination Guide:** Hướng dẫn ngắn gọn cách AI phối hợp các skill này (ví dụ: "Sử dụng Systematic Debugging kết hợp với GitNexus khi xử lý lỗi logic và đồng bộ code").
3. **Mục tiêu:** File này giúp bất kỳ AI nào khi tiếp cận Repository đều hiểu ngay bộ "vũ khí" đang có và cách sử dụng chúng tối ưu nhất cho dự án này.
### Bước 7: Thiết lập Quy tắc sử dụng Skill (RULES.md)
Để tăng tính minh bạch trong quá trình thực thi, hệ thống thực hiện cập nhật quy tắc vào file Hiến pháp:
1. Tạo (hoặc cập nhật) file `RULES.md` tại thư mục gốc dự án.
2. Bổ sung quy tắc bắt buộc: *"Khi AI sử dụng bất kỳ skill nào để tạo ra kết quả (tài liệu, hoặc câu trả lời cho người dùng...), trong kết quả đó trước tiên phải bắt đầu bằng thông báo rõ ràng về tên skill đang được sử dụng (Ví dụ: [Sử dụng Skill: concise-planning]). Ở cuối kết quả, AI phải tự rà soát lại (Self-Review) xem nội dung đầu ra đã tuân thủ hoàn toàn các hướng dẫn trong skill đó chưa."*
3. Bổ sung quy tắc Tiêu chuẩn Code: *"Luôn tuân thủ nguyên tắc Clean Code (Toàn bộ tiêu chuẩn được định nghĩa và đối chuẩn thông qua kỹ năng hệ thống: \`clean-code\`)."*
