<ai_system_protocol>
<purpose>
Tiêu chuẩn hóa quy trình cài đặt, cấu hình và kích hoạt các MCP Servers (Model Context Protocol). File này thiết kế như một kịch bản thực thi chính xác cao để mọi AI Assistant có thể chạy tự động mà không bị lỗi "ảo giác" (hallucination), không cần đoán mò đường dẫn hay cú pháp. Đây là cấu hình toàn cục (Global Setup) — chỉ cần chạy một lần cho mỗi máy tính.
</purpose>

<strict_rules>
- KHÔNG XÀI "GIẢ ĐỊNH": Không bao giờ giả định tên của một gói NPM. Chỉ sử dụng tên package đã được xác minh trong file này.
- GỌN NHẸ (NO GLOBAL INSTALL): Ưu tiên thiết lập MCP Server bằng cú pháp tự tải `npx -y` trực tiếp trong file cấu hình JSON thay vì cài đặt toàn cục (`npm install -g`), trừ khi tài liệu của tool đó bắt buộc.
- TỰ ĐỘNG HOÀN TOÀN: AI phải tự xử lý toàn bộ quy trình bao gồm cả việc restart. Không được yêu cầu người dùng thực hiện bất kỳ thao tác thủ công nào.
</strict_rules>

<execution_workflow>

### Bước 1: Quét Môi trường và Tự động Cứu hộ (Auto-Healing)
1. Dùng công cụ Command Line chạy `node --version` và `npm --version`.
2. **[Tự động Cài đặt]** NẾU không có NodeJS/NPM, AI phải tự động nhận diện Hệ điều hành và dùng công cụ CLI để cài đặt lập tức:
   - Windows: Chạy lệnh `winget install -e --id OpenJS.NodeJS`
   - MacOS: Chạy lệnh `brew install node`
   - Linux: Chạy lệnh `sudo apt update && sudo apt install nodejs npm -y`
   *(Sau khi lệnh cài đặt hoàn tất, tự động tải lại môi trường và xác minh lại bằng `node -v` rồi mới qua bước tải config MCP).*
3. Truy tìm file cấu hình MCP. Ưu tiên kiểm tra kho gốc của nền tảng Antigravity hiện tại của AI. Chạy lệnh kiểm tra file tại:
   - Windows: `$env:USERPROFILE\.gemini\antigravity\mcp_config.json`
   - MacOS/Linux: `~/.gemini/antigravity/mcp_config.json`
4. Nếu file `mcp_config.json` chưa tồn tại, hãy tự sinh ra một file JSON với cấu trúc cơ bản như sau:
   ```json
   {
     "mcpServers": {}
   }
   ```

### Bước 2: Tích hợp Cấu hình MCP (Vibe Setup)
Các MCP tool được chia làm 2 nhóm:

**🔒 Nhóm Cố định (Bắt buộc mọi dự án):**
Đây là các tool thiết yếu cho mọi quy trình vận hành. Luôn cài đặt.

```json
{
  "mcpServers": {
    "dev-tools": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-puppeteer"]
    },
    "leankg": {
      "command": "npx",
      "args": ["-y", "@smithery/cli@latest", "run", "@leankg/mcp-leankg", "--key", "YOUR_SMITHERY_KEY"]
    }
  }
}
```

> **⚠️ Lấy Smithery API Key (Bắt buộc cho LeanKG):**
> 1. Truy cập **https://smithery.ai** → Đăng nhập bằng GitHub.
> 2. Vào mục **Settings → API Keys** → Nhấn **"Create Key"**.
> 3. Copy key (dạng `sk-...`) và thay thế `YOUR_SMITHERY_KEY` trong config trên.
> 4. AI phải hỏi người dùng cung cấp key này trước khi tiếp tục — **TUYỆT ĐỐI không bỏ qua bước này**, vì LeanKG sẽ lỗi `401 Unauthorized` ngay lập tức nếu thiếu key.


**🔧 Nhóm Tùy chọn (Cài theo nhu cầu dự án):**
AI phải phân tích dự án (đọc README.md, AGENTS.md) để quyết định có cần cài hay không.

```json
{
  "mcpServers": {
    "firebase": {
      "command": "npx",
      "args": ["-y", "firebase", "experimental:mcp"],
      "điều kiện": "Cài khi dự án cần lưu trữ dữ liệu người dùng, authentication, hoặc hosting đám mây."
    },
    "google-cloud": {
      "command": "npx",
      "args": ["-y", "@google/mcp-server-cloud-run"],
      "điều kiện": "Cài khi dự án cần deploy lên Google Cloud Run hoặc quản lý hạ tầng GCP."
    }
  }
}
```
*(Ghi chú: Nếu AI phát hiện tên gói thay đổi theo phiên bản mới nhất trên NPM, lập tức cảnh báo người dùng để xin quyền thay thế bằng tên gói đúng, TUYỆT ĐỐI không điền bừa).*

### Bước 3: Restart Tự động (Auto-Restart)
Sau khi ghi file cấu hình, AI phải chủ động khởi động lại môi trường:
1. Thông báo cho người dùng: *"Cấu hình MCP đã hoàn tất. Đang yêu cầu khởi động lại để kích hoạt..."*
2. Sử dụng công cụ phù hợp để trigger reload/restart phiên AI (nếu có quyền). Nếu môi trường yêu cầu phê duyệt của người dùng (ví dụ: run_command cần approve), hãy đề xuất lệnh restart và giải thích lý do rõ ràng để người dùng confirm.
3. Sau khi confirm restart thành công, kiểm tra lại bằng cách liệt kê các MCP tool đang hoạt động.

### Bước 4: Khám phá và Soạn "Menu Công Cụ" (Tool Menu Creation)
Khi phiên làm việc được Reload:
1. AI tự động lập danh sách toàn bộ các cấu trúc/câu lệnh MCP vừa được cài đặt thành công.
2. Dựa trên chức năng của các MCP, AI tự động biên soạn một "Cuốn Menu Công Cụ" nội bộ, tuân theo định dạng sau:
   - **Tên MCP/Tool**
   - **Có gì trong tay:** (Ví dụ: "Hệ thống duyệt web ảo Puppeteer có khả năng click, chụp ảnh").
   - **Khi nào cần dùng:** (Ví dụ: "Bắt buộc dùng để mở đường dẫn localhost, chụp ảnh màn hình và tự review lỗi sau mỗi lần sửa giao diện").
3. Mục đích: Biến các server mới tải về thành một bảng phân loại rõ ràng, giúp mọi AI sau này biết chính xác "Khi mình cần làm A thì hãy lôi thẻ Tool B ra xài".

### Bước 5: Đưa Menu vào AGENTS.md
1. Mở file `AGENTS.md` (Được sinh ra từ Quy trình cài Skill số 2).
2. Dùng công cụ sửa file, tạo chuyên mục `### MCP Tools Menu` và chèn toàn bộ cuốn Menu Công Cụ vừa lập ở Bước 4 vào đây.

### Bước 6: Khởi tạo Hiến Pháp (RULES.md)
Vì các luật lệ (Iron Rules) phụ thuộc vào việc có tool nào đã được cài thành công, nên AI phải xử lý thông minh:
1. Tạo (hoặc cập nhật) file `RULES.md` tại thư mục gốc dự án.
2. Dựa vào danh sách Menu ở Bước 4, tự động đúc kết ra các "Mệnh Lệnh Sắt" (Iron Rules) ép buộc AI dùng các tool đó.
3. Các luật tiêu chuẩn cần sinh ra (Tùy theo tool):
   - NẾU cài thành công LeanKG ➔ Ghi luật: *"KHÔNG ĐƯỢC CHẠM VÀO LOGIC CỐT LÕI nếu chưa dùng LeanKG quét Blast Radius."*
   - NẾU cài thành công Puppeteer ➔ Ghi luật: *"KHÔNG ĐƯỢC KẾT THÚC TASK UI nếu chưa dùng Puppeteer chụp và tự kiểm định."*
   - NẾU cài thành công Firebase/GCP ➔ Ghi luật: *"MỌI THAO TÁC DATA/CLOUD đều phải chạy tự động qua MCP, tuyệt đối không bắt người dùng tự gõ lệnh."*
4. Thông báo quá trình hoàn tất cài đặt toàn bộ "Tay chân và Hiến pháp" cho người dùng.
