<ai_system_protocol>
<purpose>
Tiêu chuẩn hóa quy trình chuyên môn hóa nhóm làm việc hướng tới mô hình Multi-agent (System-level Sub-agents). Thay vì chỉ "nhập vai" ảo, file này hướng dẫn AI khởi tạo các Agent như những "thực thể độc lập" ở cấp hệ thống. Nhờ đó, người dùng có thể trực tiếp giao việc theo đúng chuyên môn, giúp giảm tải nơ-ron và ép AI chỉ hoạt động trong đúng phạm vi kỹ năng được giao.
</purpose>

<strict_rules>
- ĐỘC LẬP TUYỆT ĐỐI KHÔNG GIAO THOA: Không gán kỹ năng thẩm mỹ cho Core hay kỹ năng logic cho UX.
- PHẢI TẠO THƯ MỤC VẬT LÝ: AI bắt buộc phải khởi tạo các agent vào thư mục hệ thống để môi trường IDE có thể nhận diện chúng như những Skill thực thụ.
- PHẢI SHOW PROMPT MẪU: Sau khi setup, AI phải in ra các đoạn lệnh để người dùng hiểu cách gọi Agent.
</strict_rules>

<execution_workflow>

### Bước 1: Khởi tạo Cấu trúc Thực thể (System-level Setup)
Để IDE (như Antigravity/Claude Dev) ghi nhận sự hiện diện thực sự của các nhân sự, AI tự động dùng lệnh để tạo cấu trúc thư mục đặc biệt tại gốc dự án (Tên thư mục phải bắt buộc có tiền tố `A-` ở đầu):
- Tạo thư mục Agent Lõi tại: `.claude/skills/agents/A-core/`
- Tạo thư mục Agent Thẩm mỹ tại: `.claude/skills/agents/A-ux/`
- Tạo thư mục Agent Kiểm thử tại: `.claude/skills/agents/A-test/`

### Bước 2: Nạp Linh hồn và Phân bổ Tài nguyên
Trong mỗi thư mục Agent vừa tạo, AI khởi tạo file cấu hình mang tên cố định là `SKILL.md` (viết hoa toàn bộ) để "thổi hồn" vào.
File cấu hình `SKILL.md` của mỗi Agent phải chứa rõ 2 thứ:
1. **Persona & Function**: Định nghĩa rõ họ là ai. Core lo logic hiệu suất, UX lo thẩm mỹ giao diện, Test lo tìm bug.
2. **Skill & Tool Binding**: Chỉ định chính xác các kỹ năng/công cụ họ được phép gọi qua đường dẫn tương đối hoặc nhắc tên, ví dụ:
   - **Nội dung file `SKILL.md` của @A-Core**: Nối với `javascript-mastery`, `modern-javascript-patterns`, `clean-code`, `game-development`.
   - **Nội dung file `SKILL.md` của @A-UX**: Nối với `antigravity-design-expert`, `ui-ux-pro-max`, `modern-javascript-patterns`.
   - **Nội dung file `SKILL.md` của @A-Test**: Nối với `systematic-debugging`, `lint-and-validate`, cấu hình để dùng tool `leankg` quét rủi ro và `puppeteer` (dev-tools) tự chụp ảnh màn hình.

### Bước 3: Cập nhật Sơ đồ Nhân sự (AGENTS.md)
1. Mở file `AGENTS.md` ở gốc dự án.
2. Cập nhật nhóm 3 Agent chuyên biệt này vào danh sách.
3. Giải thích ngắn gọn vai trò của họ để phục vụ việc tra cứu của người dùng.

### Bước 4: Tạo "Bảng Lệnh Triệu Hồi" (Sample Prompts)
AI phải tiến hành phân tích sâu mã nguồn và mục tiêu của dự án để tự động soạn thảo 3 khối code (Markdown code blocks) chứa các lệnh thực thi thực tế. Tuyệt đối không dùng các ví dụ chung chung mà phải dựa trên những gì dự án đang thực sự cần:

1. **Prompt cho @A-Core (Phát triển)**: Dựa trên code hiện tại, hãy gợi ý phát triển thêm một tính năng logic gọn nhẹ, thiết yếu mà dự án còn thiếu.
2. **Prompt cho @A-UX (Cải tiến)**: Dựa trên giao diện hiện tại, hãy gợi ý một cải tiến về thẩm mỹ, hiệu ứng hoặc trải nghiệm người dùng để nâng tầm sản phẩm.
3. **Prompt cho @A-Test (Kiểm chứng)**: Gợi ý một quy trình kiểm tra toàn diện, yêu cầu sử dụng kết hợp LeanKG để rà lỗi logic mã nguồn và Puppeteer để kiểm định hiển thị trên trình duyệt.

### Bước 5: Xác nhận Hoàn tất
1. In danh sách 3 bộ Prompt Mẫu đã được cá nhân hóa cho dự án lên khung chat.
2. Thông báo dõng dạc: "Các thực thể Agent Hệ thống đã được khởi tạo và tối ưu hóa riêng cho dự án này! Hãy Copy các lệnh trên để bắt đầu 'Vibe Coding' ngay."
### Bước 6: Thiết lập Quy tắc định danh Agent (RULES.md)
Để nâng cao tính chuyên nghiệp và minh bạch trong mô hình Multi-agent, AI cập nhật file Hiến pháp:
1. Mở file `RULES.md` tại thư mục gốc dự án.
2. Bổ sung quy tắc bắt buộc: *"Khi một sub-agent được gọi (Core, UX, Test...), trong câu trả lời hoặc sản phẩm đầu ra của agent đó trước tiên luôn phải bắt đầu bằng tên định danh của mình (Ví dụ: [@A-Core], [@A-UX]...). Ngoài ra, nếu bất kỳ thay đổi nào tác động đến kiến trúc hệ thống, Agent bắt buộc phải cập nhật đồng thời file ARCHITECTURE.md và file README.md."*
